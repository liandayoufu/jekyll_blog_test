git 知识要点

git 是一个去中心化的版本管理工具，不需要时时在线，只有在开发完成提交时在线即可，不同于svn需要时时在线。

###

bug 分支管理

场景：正在dev 分支做开发，但是还没有完成，暂时还不能提交。

  • git stash 临时保存工作现场
  • git checkout master 假定是需要在master分支做修复，切换到master分支
  • git checkout -b issue-101 创建新的issue-101bug修复分支
  • git commit -m "bug 101 fixed" 修复完成提交
  • git checkout master; git merge -m "merged bug fixed 101" issue-101 切换到master并将issue-101分支合并过来，
    • –no-ff 参数用普通方式合并，合并后的历史有分支，能看出来曾经做过合并，fast forward合并看不出来曾经做过合并
  • git stash list 可以查看之前保存的stash信息
  • git stash apply 可以恢复现场，但是恢复后stash内容不删除，需要git stash drop 删除；git stash pop 相当于前面两条命令

Tips

关于 .gitigore
如果已经提交 需要先 git rm -r --cache .
然后再 git add .

git log 显示直观些
git log --graph --all --pretty=format:'%Cred%h%Creset -%C(yellow)%d%Creset %s %Cgreen(%cr) %C(bold blue)<%an>%Creset' --abbrev-commit --date=relative
详细格式用法man git-log查看

可以配置别名
git config --global alias.lg "log --graph --all --pretty=format:'%Cred%h%Creset -%C(yellow)%d%Creset %s %Cgreen(%cr) %C(bold blue)<%an>%Creset' --abbrev-commit --date=relative"
之后用git lg 即可

随便说说

发表于 2016-01-06   |     |   阅读次数

每每听到看到医院里面人满为患，各种不治之症，各种英年早逝。在科学技术日益发达的今天，却不见人们越少受于疾病之折磨，伴之而来 的却是越来越多的病名，越来越多的无法治愈之病名，越来越多的束手无策。

鸣呼哀哉！

想我大中华之中医历经几千年亿万生命的验证，以亿万生命血的代价换来的经验，中华传统文化之瑰宝也沦落到如此地步，真是痛哉！普遍崇尚西医，大家多以为西方西学是现代科学技术比较先进，见效快，中医疗效慢，只能调理，不能治病，各大中医院也是如此，以各种中西医结合为名头，不管看啥，进了中医院也一色先一堆检查拿了检查报告再说，仿佛没了报告就看不了病一样。殊不知，在古时，中国人没有这些个先进的仪器，看病效果竟是那样之好，诞生出那么多名医，那么多杏林佳话。而古时之所以会有那么多的重病，往往是因为当时信息交通诸多不便，求医不若现在那么方便。著名物理学家钱学森也对中医与西医有过评价，中医实乃科学，而西方医学充其量叫做科技。中医一个人往这一站就是一个医院了，这不是吹牛叉，这缘于中医对人体生理学的深刻理解和千万生命和岁月带来的经验累积。

一直对中医深信不疑，无奈一直处于门外徘徊状态，终不得一窥究竟。更何况于对中医一无所知的人，定然是难以对中医树立起坚强的信心。但兵法有云，知已知彼方可百战百胜

你根本都不曾了解过中医，不知中医治病之原理，凭什么说中医是骗人的？中医不可信？

git 知识要点

git 是一个去中心化的版本管理工具，不需要时时在线，只有在开发完成提交时在线即可，不同于svn需要时时在线。

###

bug 分支管理

场景：正在dev 分支做开发，但是还没有完成，暂时还不能提交。

  • git stash 临时保存工作现场
  • git checkout master 假定是需要在master分支做修复，切换到master分支
  • git checkout -b issue-101 创建新的issue-101bug修复分支
  • git commit -m "bug 101 fixed" 修复完成提交
  • git checkout master; git merge -m "merged bug fixed 101" issue-101 切换到master并将issue-101分支合并过来，
    • –no-ff 参数用普通方式合并，合并后的历史有分支，能看出来曾经做过合并，fast forward合并看不出来曾经做过合并
  • git stash list 可以查看之前保存的stash信息
  • git stash apply 可以恢复现场，但是恢复后stash内容不删除，需要git stash drop 删除；git stash pop 相当于前面两条命令

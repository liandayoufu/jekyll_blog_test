Windows主机添加自定义服务

发表于 2016-04-18   |   分类于 windows   |     |   阅读次数

原文地址：http://www.cnblogs.com/wangchenxicool/articles/2553966.html

实现其开机自动启动步骤其实很简单。这里用到的是微软的两个小工具。instsrv.exe和srvany.exe

微软对instsrv.exe的官方说明如下：

Installs and uninstalls executable services and assigns names to them. 也就是个加载services的小东东。

其用法如下：instsrv （这里的srvany path也就是工具srvany.exe的路径了）

解释下，srvany.exe是微软出的用于将一个程序注册为一个服务的小程序。它可以实现讲任何程序设置成服务启动。具体用法可以自己想下哦。

解释完毕，现在来实际的操作了吧。

1.讲所需要的工具instsrv.exe和srvany.exe放在一个文件夹内，在这我放在d:\tools中吧。

2.需要了解vmware.exe的安装路径，以本机为例：D:\VMware Workstation\vmware.exe
要启动的虚拟机配置文件路径，我的2003虚拟机的配置文件windows2003.vmx的路径是D:\vmare\windows2003\windows server 2003.vmx

3.新建服务

假设服务名为vm_autostart，所以我的命令行是：

instsrv VM_AutoStart D:\tools\srvany.exe

4.注册服务

在注册表中，定位到HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\vm_autostart
新建项：”Parameters”
在”Parameters”项里面，新建字符串”Application”，
字符串的值：
“D:\VMware Workstation\vmware.exe” -x “D:\vmare\windows2003\windows server 2003.vmx”

(如果你想要开机加载两个虚拟机系统，只需要在其后再添加一个虚拟机配置文件路径即可）

吴述重订伤寒论条文背诵（太阳篇）

太阳主证

病发于阳，病发于阴

  • 1、病有发热恶寒者，伤于阳也，无热恶寒者，伤于阴也，伤于阳者七日愈，伤于阴者六日愈，以阳数七，阴数六故也。(7)
  • 2、病人身大热，反欲得衣者，寒在皮肤，热在骨髓也；身大寒，反不欲近衣者，寒在皮肤，热在骨髓也。(11)

脉证提纲

  • 3、太阳病，脉浮，头项强痛而恶寒。(1)
  • 4、太阳病，发热，汗出，脉阴阳俱浮者，名为中风。(2)
  • 5、太阳病，或已发热，或未发热，必恶寒，体痛，呕逆，脉阴阳俱紧者，名为伤寒。(3)
  • 6、太阳病，发热而渴，不恶寒者，为温病；若发汗已，身灼热者，名风温。风温为病，脉阴阳俱浮，自汗出，多眠睡，鼻息必鼾，语言难出；若被下之，小便不利，直视失溲；若被火者，微发黄色，剧则如惊痫，时瘛(chi)疭(zong)，若火熏之。一逆尚引日，再逆促命期。(6)

太阳传经

  • 7、伤寒一日，太阳受之，脉若静者，为不传；颇欲吐，若躁烦，脉数急者，为传也。(4)
  • 8、伤寒二三日，阳明、少阳证不见者，为不传。(5)
  • 9、太阳病，病至七日以上自愈者，以行其经尽故也。若欲再传其经，针足阳明，使经不传则愈。(8)
  • 10、太阳病，欲解时，从巳至未上。(9)
  • 11、风家，表解而不了了者，十二日愈。(10)
  • 12、凡病若发汗，若吐，若下，若亡血、亡津液，阴阳自和者，必自愈。(58)
  • 13、大下之后，复发汗，小便不利，亡津液故也。勿治之，得小便利，必自愈。(59)

升级Win10时语言不同不能保留apps

发表于 2016-02-16   |   分类于 Windows   |     |   阅读次数

公司一笔记本升级win10,自带的是OEM的中文版win7 Home Basic，因为需要用英文版，anytime upgrade到Ultimate版。现在下好了英文版win10的安装包，那么问题来了，到了选择保留哪些东西的那一步卡壳了，keep personal settings and apps那项不能选，下面有一句 ”You can‘t keep Windows settings, personal files, and apps because you’ve chosen to install Windows 10 using a different language than you're currently using. “
造成这个问题是因为虽然你把系统显示语言改成英文了但是UI language依然是中文，运行dism /online /get-intl 可以看到

解决办法：

  • 重启，按F8, 选 Repair Your Computer –> Command Prompt
  • ` dism /image:e:\ /set-uilanguage:en-US （e是你系统所在的那个盘符，修复模式其实是启动PE系统，所以系统盘符会变，这个在你选择命令行模式的那个界面有显示，看一下就好了）
  • 再重启就可以正常安装了，微软系统从win8开始语言包就可以随便改了，家庭版也无妨。不像win7必须得是旗帜版才可以改系统语言，专业版都不好使。

git 知识要点

git 是一个去中心化的版本管理工具，不需要时时在线，只有在开发完成提交时在线即可，不同于svn需要时时在线。

###

bug 分支管理

场景：正在dev 分支做开发，但是还没有完成，暂时还不能提交。

  • git stash 临时保存工作现场
  • git checkout master 假定是需要在master分支做修复，切换到master分支
  • git checkout -b issue-101 创建新的issue-101bug修复分支
  • git commit -m "bug 101 fixed" 修复完成提交
  • git checkout master; git merge -m "merged bug fixed 101" issue-101 切换到master并将issue-101分支合并过来，
    • –no-ff 参数用普通方式合并，合并后的历史有分支，能看出来曾经做过合并，fast forward合并看不出来曾经做过合并
  • git stash list 可以查看之前保存的stash信息
  • git stash apply 可以恢复现场，但是恢复后stash内容不删除，需要git stash drop 删除；git stash pop 相当于前面两条命令
